Data Layer — Firestore Schema

Two documents per user. Simple rules. Zero data loss on migration.

💡 Why one document?

The app always reads and writes its full state at once — just like it does with localStorage today. One appdata document keeps the code changes minimal. Firestore's 1 MB limit comfortably holds ~5,000+ transactions.

🔒 Security Rules
match /users/{uid}/{d=**} {
 allow read, write:
  if request.auth.uid == uid;
}

Three lines. Users can only touch their own documents. No exceptions.

🔑 API Key Flow

User pastes their Anthropic key once in Settings. Saved to their profile doc. On AI Parse: app reads key from memory → sends as X-Api-Key header → Netlify Function forwards to Anthropic. Key never logged server-side.
